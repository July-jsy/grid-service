# 街道办网格化便民服务管理系统

---

## 快速启动

### 环境要求

- JDK 17+
- Maven 3.9+
- Node.js 20.19+
- npm 10+

### 启动后端

在项目根目录执行：

```powershell
cd backend
mvn spring-boot:run
```

后端默认地址：`http://localhost:8080`

### 启动前端

新开一个终端，在项目根目录执行：

```powershell
cd frontend
npm install
npm run dev
```

前端默认地址：`http://localhost:5173`

### 登录账号

系统采用**管理员 / 普通用户**两级角色，角色定位如下：

- **管理员（街道办/社区工作人员）**：负责后台管理与业务办理，拥有系统全部功能
- **普通用户（辖区居民）**：通过登录页自助注册，用于在线办事和事件上报，不可访问后台管理功能

| 角色 | 用户名 | 密码 | 可访问功能 |
|------|--------|------|------------|
| 管理员 | `admin` | `admin123` | 全部功能（见下方"管理员功能"） |
| 普通用户 | `user` | `user123` | 仅居民服务功能（见下方"普通用户功能"） |

#### 管理员功能

| 模块 | 说明 |
|------|------|
| 数据总览 | 查看辖区关键指标卡片、事件状态统计、处置率汇总 |
| 网格管理 | 网格信息增删改查、网格与社区楼栋层级关联 |
| 基础信息采集 | 居民档案管理（常住/流动/特殊人群）、房屋档案管理 |
| 便民服务管理 | 服务事项发布与管理、申请受理与状态流转 |
| 事件处置 | 事件筛选分派、处置状态流转（待受理→处理中→已办结）、核查结案 |
| 用户管理 | 系统用户新增、查询和删除 |

#### 普通用户功能

| 模块 | 说明 |
|------|------|
| 在线申办 | 浏览服务事项、填写表单提交申请 |
| 进度查询 | 实时查看申请办理状态（待受理→办理中→已办结） |
| 事件上报 | 提交事件（含文字描述和图片上传）、查看自己上报的事件处理进度 |
| 通知公告 | 查看社区发布的停水停电、活动通知、政策宣传等信息 |

> **设计说明**：普通用户通过登录页自助注册，角色定位为辖区居民，因此只开放居民自助服务功能。网格管理、基础信息采集、事件处置等内部办公功能仅限管理员使用。

### 当前已实现功能

**公共功能（管理员 & 普通用户均可使用）：**
- Session 登录、退出、角色权限及接口登录校验
- 登录页普通用户自助注册，注册密码使用 BCrypt 加密存储
- 响应式 PC Web 管理界面

**管理员专属功能：**
- 数据总览与事件状态统计
- 网格信息增删改查
- 居民档案和房屋档案管理
- 服务事项管理、申请受理与状态流转
- 事件筛选分派与处置状态流转
- 系统用户新增、查询和删除

**普通用户专属功能：**
- 在线申办：浏览服务事项、填写表单提交申请
- 进度查询：查看申请办理状态
- 事件上报：提交事件信息、跟踪处理进度

- 通知公告：查看社区通知、便民服务和政策宣传信息

**权限与数据隔离：**
- 普通用户只能查看自己的服务申请和上报事件

> 当前版本为可演示 MVP，后端使用 MyBatis-Plus + SQLite 持久化业务数据。首次启动会自动创建 `backend/grid-service.db` 并写入示例数据。GIS、Excel 导出等内容属于后续演进设计。

### SQLite 是否需要安装

不需要单独安装 SQLite 数据库服务。后端通过 MyBatis-Plus 和 Maven 依赖 `org.xerial:sqlite-jdbc` 直接读写 SQLite 文件：

```xml
<dependency>
    <groupId>org.xerial</groupId>
    <artifactId>sqlite-jdbc</artifactId>
    <version>3.49.1.0</version>
</dependency>
```

执行 `mvn spring-boot:run` 时 Maven 会自动下载驱动，系统会自动创建 `backend/grid-service.db`。如需手工查看数据库内容，可选装 DB Browser for SQLite，但它不是项目运行的必要条件。

## 一、系统功能模块设计

### 1.1 功能模块总览

本系统规划为**五大核心模块**，各模块相互关联、数据互通，形成完整的服务体系。

```
┌─────────────────────────────────────────────────────────────┐
│                  街道办网格化便民服务管理系统                      │
├──────────┬──────────┬──────────┬──────────┬──────────────────┤
│  网格管理  │  便民服务  │  事件处置  │  数据大屏  │  系统管理         │
│  模块     │  模块     │  模块     │  模块     │  模块            │
├──────────┴──────────┴──────────┴──────────┴──────────────────┤
│              PC Web端（响应式布局，兼容移动浏览器）                │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 网格管理模块

#### 1.2.1 网格划分与维护
- 网格信息管理：网格编号、名称、范围描述、所属社区
- 支持网格的增、删、改等基本操作
- 网格与社区、楼栋的层级关联
- 可扩展：后期接入高德/天地图实现GIS可视化

#### 1.2.2 网格员管理
- 网格员信息档案管理（基本信息、联系方式、负责网格区域）
- 网格员与网格的绑定关系管理
- 网格员工作统计（任务完成数、事件处理时效、居民评价）

#### 1.2.3 基础信息采集
- **人口信息**：常住人口、流动人口、特殊人群（低保户、残疾人、独居老人、重点青少年等）分类管理
- **房屋信息**：房屋坐落、产权类型、出租状态、居住人员关联
- **组织信息**：辖区单位、社会组织、商户门店基础信息
- **基础设施**：道路、绿化、消防设施、监控点位等信息登记

### 1.3 便民服务模块

#### 1.3.1 服务事项清单（示例）
| 类别 | 具体事项 |
|------|----------|
| 民政服务 | 低保申请、临时救助、高龄津贴 |
| 社保服务 | 医保参保、养老保险办理 |
| 卫健服务 | 生育登记、老年健康管理 |
| 城管服务 | 门头备案、占道审批 |
| 住房服务 | 保障房申请、物业投诉 |

#### 1.3.2 在线办事功能
- **服务指南**：办事流程、所需材料、办理时限一目了然
- **在线申办**：填写表单、上传附件材料、提交申请
- **进度查询**：实时查看申请办理状态（待受理→办理中→已办结）
- **预约办理**：线上预约线下窗口办理时间

#### 1.3.3 信息发布与互动
- 通知公告发布（停水停电、活动通知、政策宣传）
- 社区活动发布与报名
- 居民意见反馈与留言

### 1.4 事件处置模块

#### 1.4.1 事件来源
- 居民上报（通过系统网页端提交，含文字描述和图片上传）
- 网格员巡查发现（网格员主动录入）
- 上级交办/其他部门转办

#### 1.4.2 事件分类
| 一级分类 | 二级分类示例 |
|----------|-------------|
| 城市管理 | 占道经营、违章搭建、井盖缺失 |
| 公共安全 | 消防隐患、治安线索 |
| 环境卫生 | 垃圾堆积、噪音扰民、小广告 |
| 矛盾纠纷 | 邻里纠纷、物业矛盾 |
| 民生服务 | 水电故障、特殊人群求助 |

#### 1.4.3 处置流程

```
上报/发现 → 自动派单(依据网格归属) → 网格员响应 → 现场处置
                                                    ↓
                                            处置完成(拍照上传)
                                                    ↓
                                    ┌──────────────────────┐
                                    │  核查结案/退回重办      │
                                    │  (社区负责人或值班长)   │
                                    └──────────────────────┘
                                                    ↓
                                              居民评价
```

- **时限预警**：黄牌提醒（即将超期）、红牌警告（已超期）
- **升级机制**：网格员无法处置时，可逐级上报至社区→街道→区级
- **催办督办**：超期未处理案件自动推送至上级领导督办

### 1.5 数据大屏模块

#### 1.5.1 综合数据大屏
- 关键指标卡片呈现（今日事件数、处置率、办结率、居民满意度）
- 事件分类统计图（饼图/柱状图）、趋势变化曲线（折线图）
- 各社区/网格排名一览

#### 1.5.2 统计分析与报表
- **事件分析**：事件类型分布（饼图）、趋势变化（折线图）、各社区对比（柱状图）
- **效能分析**：平均处置时长、办结率、超期率统计
- **服务分析**：高频服务事项排名、满意度统计
- **数据导出**：支持Excel导出

### 1.6 系统管理模块

- **用户与权限管理**：基于角色的权限控制，分为**管理员**和**普通用户**两级角色：
  - **管理员（街道办/社区工作人员）**：可访问全部功能模块，包括网格管理、基础信息采集、便民服务管理、事件处置、数据总览、用户管理。管理员账号由系统初始化创建，不可自助注册。
  - **普通用户（辖区居民）**：通过登录页自助注册，仅可访问居民服务功能（在线申办、进度查询、事件上报、通知公告）。不可访问后台管理功能（网格管理、基础信息采集、事件处置、用户管理等）。
- **数据字典配置**：事件类型、服务事项分类等系统参数可配置
- **操作日志记录**：关键操作留痕（登录、事件处置、数据修改）

## 二、技术架构设计

### 2.1 总体架构

采用**B/S单体架构**，前后端分离，结构简单、易于开发和部署，适合小组快速交付原型系统。

```
┌───────────────────────────────────────────────────┐
│                    前端展示层                        │
│   PC管理端 (Vue3 + Element Plus)  │  数据大屏       │
│                                   │  (ECharts)     │
├───────────────────────────────────────────────────┤
│                    后端服务层                        │
│          Spring Boot 单体应用 (RESTful API)         │
│    ┌──────────────────────────────────────────┐    │
│    │  网格管理 │ 便民服务 │ 事件处置 │ 数据统计 │    │
│    └──────────────────────────────────────────┘    │
├───────────────────────────────────────────────────┤
│                    数据存储层                        │
│               SQLite + 本地文件存储                    │
└───────────────────────────────────────────────────┘
```

### 2.2 技术选型

| 层级 | 技术选型 | 版本 | 说明 |
|------|----------|------|------|
| 前端框架 | Vue 3 | 3.5.35 | 组合式 API，响应式框架 |
| UI 组件库 | Element Plus | 2.14.1 | 组件丰富，快速搭建后台管理界面 |
| 构建工具 | Vite | 7.3.x | 新一代前端构建工具，HMR 极速热更新 |
| 状态管理 | Pinia | 3.0.4 | Vue 3 官方状态管理库 |
| HTTP 客户端 | Axios | 1.17.0 | 封装请求拦截、统一响应处理 |
| 图表可视化 | ECharts | 6.1.0 | 数据大屏图表渲染 |
| 后端框架 | Spring Boot | 3.5.14 | 单体应用，约定优于配置 |
| ORM 框架 | MyBatis-Plus | 3.5.12 | 通过 BaseMapper 简化 SQLite 数据访问 |
| 数据库 | SQLite (sqlite-jdbc) | 3.49.1.0 | 轻量级文件数据库，零配置，无需安装服务，适合原型演示 |
| 文件存储 | 本地文件系统 | — | 图片、附件存储 |
| 认证授权 | Session（Servlet 内置） | — | 基于 Cookie 的会话认证，零额外依赖 |
| 密码加密 | BCrypt (Spring Security) | — | 密码加密存储 |
| 构建工具 | Maven | 3.9.x | Java 项目构建与依赖管理 |
| 运行环境 | JDK / Node.js | JDK 17 / Node.js 20.19+ | 后端/前端运行时 |

### 2.3 技术选型说明

考虑到本项目为三人小组作业（原型系统），技术选型遵循以下原则：

- **够用即可**：单体架构满足功能演示需求，不引入微服务、消息队列、分布式缓存等复杂组件
- **零部署成本**：SQLite 为文件型数据库，无需安装数据库服务，项目启动即用；答辩时拷贝一个文件即可迁移全部数据
- **快速上手**：选用组员熟悉的 Java/Vue 技术栈，降低学习成本
- **易于部署**：单体应用一键启动，无需容器化编排；前端静态资源可直接放在 Spring Boot 中打包部署
- **可演进性**：代码按模块分包，后续如需拆分微服务，包结构可直接映射为独立服务

### 2.4 数据安全设计（原型阶段）

- 居民隐私数据脱敏处理（身份证号、手机号部分隐藏）
- 用户密码加密存储（BCrypt）
- 接口权限校验（不同角色访问不同功能）
- 关键操作日志记录
